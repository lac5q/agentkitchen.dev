# Phase 35 Plan 04: A2A Delegation, ADK Proof, And Registry/Flow Surfacing

<objective>
Complete the end-to-end Phase 35 proof: Kitchen delegates to registered A2A agents, an ADK-shaped fixture can register and accept a task, and Registry/Flow surfaces A2A/ADK agents safely from the canonical roster.
</objective>

<threat_model>
## Threat Model

- Threat: outbound delegation sends bearer credentials to the wrong remote endpoint or logs configured secrets. Severity: high. Mitigation: credentials are read from safe metadata/config only, never displayed in UI, never included in test snapshots, and outbound client tests assert redaction.
- Threat: UI exposes full URLs with embedded credentials, API keys, or auth headers. Severity: high. Mitigation: URL display helper redacts credentials, drawer/table tests assert no `Authorization`, `Bearer `, or `ak_` secret strings are rendered.
- Threat: ADK proof becomes required infrastructure for every developer or CI run. Severity: medium. Mitigation: CI uses `examples/adk-a2a-agent/agent-card.json` and a mocked A2A server; `agent.py` and README are optional runnable proof instructions.
- Threat: Flow reintroduces hardcoded ADK or agent names instead of using the canonical registry. Severity: medium. Mitigation: tests register an ADK-shaped A2A agent and assert it appears dynamically by registry ID.
</threat_model>

## Task B - Implement A2A outbound client and dispatch adapter

<read_first>
- `apps/kitchen/src/lib/a2a/client.ts`
- `apps/kitchen/src/lib/a2a/types.ts`
- `apps/kitchen/src/lib/a2a/config.ts`

- `apps/kitchen/src/lib/dispatch/types.ts`
- `apps/kitchen/src/lib/agent-registry.ts`
</read_first>

<action>
Create `apps/kitchen/src/lib/a2a/client.ts` exporting:

- `sendMessageToA2aAgent(agent, task, options?)`
- `getRemoteA2aTask(agent, taskId, options?)`
- `cancelRemoteA2aTask(agent, taskId, options?)`
- `redactUrlForDisplay(rawUrl: string): string`

Concrete behavior:
- Endpoint URL is read from `agent.metadata.a2a.endpointUrl` first, then `agent.tunnelUrl`, then `http://${agent.host}:${agent.port}`.
- Outbound send path is `${endpointUrl}/message:send` unless metadata has `a2a.messageSendUrl`.
- Timeout uses `getA2aConfig().remoteCardTimeoutMs`.
- Outbound payload includes `task_id`, `context_id`, `message`, and `metadata.kitchenDelegated: true`.
- `redactUrlForDisplay("https://user:pass@example.com/path")` returns `https://example.com/path`.

Create `apps/kitchen/src/lib/dispatch/a2a-adapter.ts` exporting `a2aAdapter` that implements `AgentAdapter` and calls `sendMessageToA2aAgent()`.

Update `apps/kitchen/src/lib/dispatch/adapter-factory.ts` so A2A protocol agents select `a2aAdapter` before platform fallback. If the existing `RemoteAgentConfig` type cannot carry `protocol`, extend `RemoteAgentConfig` with optional `protocol?: AgentProtocol` in `apps/kitchen/src/types/index.ts`.
</action>
