# Phase 35 Plan 02: A2A Agent-Card Ingestion And Canonical Registration

<objective>
Implement A2A remote-agent discovery as agent-card ingestion into the Phase 34 canonical registry, with SSRF-aware fetching, card validation, normalized capabilities, and ADK-shaped fixture coverage.
</objective>

<threat_model>
## Threat Model

- Threat: arbitrary agent-card URL fetching becomes SSRF against local files, metadata IPs, or internal services. Severity: high. Mitigation: reject non-HTTP(S) schemes, reject URLs with credentials, reject metadata IPs `169.254.169.254` and `metadata.google.internal`, enforce timeout/size limits, and make private-network allowance explicit via `KITCHEN_A2A_ALLOW_PRIVATE_NETWORK_CARDS`.
- Threat: malicious remote cards spoof an existing canonical agent or inject unsafe metadata. Severity: high. Mitigation: deterministic default agent ID from `sha256(cardUrl).slice(0, 12)`, duplicate detection, strict metadata allowlist under `metadata.a2a`, and route tests for duplicate identity handling.
- Threat: A2A registration bypasses Phase 34 registry/auth and creates a second source of truth. Severity: high. Mitigation: `ingestA2aAgentCard()` must call `registerAgent({ protocol: "a2a" })`; no direct `registered_agents` inserts in card-ingestion code.
</threat_model>

<action>
Create `apps/kitchen/src/lib/a2a/card-ingestion.ts` exporting:

- `function validateA2aAgentCard(value: unknown): A2aAgentCard`
- `function isAllowedAgentCardUrl(rawUrl: string, config = getA2aConfig()): boolean`
- `async function fetchA2aAgentCard(cardUrl: string, config = getA2aConfig()): Promise<A2aAgentCard>`
- `async function ingestA2aAgentCard(input: { cardUrl: string; requestedId?: string; issueApiKey?: boolean; source?: "adk" | "a2a" | "manual" }): Promise<RegisterAgentResult>`

Concrete implementation requirements:
- `fetchA2aAgentCard` uses `AbortSignal.timeout(config.remoteCardTimeoutMs)`.
- `fetchA2aAgentCard` rejects responses over 256 KiB by checking `content-length` when present and by slicing read text to a maximum of 262144 bytes.
- `validateA2aAgentCard` requires `name`, `url`, `version`, and at least one skill/capability.
- `ingestA2aAgentCard` calls `registerAgent()` exactly once for successful ingestion.
- `registerAgent()` input uses `protocol: "a2a"`.
- Platform detection uses `platform: "gemini"` when `input.source === "adk"` or the card metadata/extensions contain `adk`; otherwise use `platform: "openclaw"` as the generic A2A-compatible fallback until a broader enum exists.
- Capability IDs come from card skill IDs; names/descriptions come from card skills; tags include `"a2a"` and `"adk"` when source is ADK.
- `metadata.a2a.validationStatus` is exactly `"validated"` on success.
</action>

- Accept `POST` JSON body `{ "cardUrl": string, "requestedId"?: string, "source"?: "adk" | "a2a" | "manual", "issueApiKey"?: boolean }`.

- The returned agent appears in `GET /api/agents` because registration wrote through the canonical registry.
