# Phase 35 Plan 03: Durable A2A Task API And SSE Streaming

<objective>
Implement authenticated, durable A2A task lifecycle operations with current HTTP+JSON route names, SSE streaming, task lookup fallback, and a lightweight JSON-RPC compatibility dispatcher for non-streaming method calls.
</objective>

<threat_model>
## Threat Model

- Threat: unauthenticated clients submit, read, stream, or cancel tasks. Severity: high. Mitigation: every task route calls `authenticateAgentHeaders()` before service operations; route tests cover missing/invalid bearer keys.
- Threat: a valid agent reads or cancels another caller's private task by guessing IDs. Severity: high. Mitigation: task service filters by authenticated caller/target authorization and returns not-found-style errors without leaking existence.
- Threat: streaming connection loss loses task state or final result. Severity: medium. Mitigation: persist task and task events before streaming; `GET /tasks/{id}` remains the durable fallback.
- Threat: task payloads with sensitive or blocked content are persisted and dispatched. Severity: high. Mitigation: reuse `scanContent()` before task creation and write audit records for blocked/flagged content.
</threat_model>

CREATE TABLE IF NOT EXISTS a2a_tasks (
  task_id TEXT PRIMARY KEY,
  context_id TEXT NOT NULL,
  caller_agent_id TEXT NOT NULL,
  target_agent_id TEXT,
  state TEXT NOT NULL CHECK(state IN ('submitted','working','input-required','completed','failed','canceled')),
  message_json TEXT NOT NULL,
  artifacts_json TEXT NOT NULL DEFAULT '[]',
  metadata_json TEXT NOT NULL DEFAULT '{}',
  created_at TEXT NOT NULL,
  updated_at TEXT NOT NULL,
  terminal_at TEXT,
  cancel_requested_at TEXT

```

CREATE TABLE IF NOT EXISTS a2a_task_events (
  id INTEGER PRIMARY KEY,
  task_id TEXT NOT NULL,
  sequence INTEGER NOT NULL,
  event_type TEXT NOT NULL,
  payload_json TEXT NOT NULL,
  created_at TEXT NOT NULL,
  UNIQUE(task_id, sequence),
  FOREIGN KEY(task_id) REFERENCES a2a_tasks(task_id) ON DELETE CASCADE
);
```
