# Phase 74 Plan 01: Security Label Schema + Raw Vault

## Product Goal

Make Memroos safe to use as a memory hub for sensitive agent work by moving raw ingested message evidence into an append-only vault and stamping all legacy memory/audit tables with fail-closed security labels.

## Scope

1. Add security label dimensions to `messages`, `audit_log`, `hive_actions`, `agent_memory_writes`, and `recall_log`.
2. Add `raw_artifacts` and `artifact_labels` schema for vault metadata and artifact label versioning.
3. Implement a filesystem vault writer/reader under `~/.memroos/vault/<tenant>/<YYYY>/<MM>/<DD>/`.
4. Wire message ingestion to write newly inserted session rows to the vault.
5. Add admin-only API routes to list vault artifacts and replay a hash-verified artifact.
6. Cover schema, vault replay, ingest, and admin API behavior with tests.

## Verification Plan

- RED: targeted Vitest tests for schema labels, vault write/replay, message ingestion, admin list, and admin replay.
- GREEN: implement additive schema, vault writer/reader, ingest hook, and routes.
- Gate: run targeted Vitest suite, `npm --prefix apps/memroos run typecheck`, and `npx gitnexus detect-changes --repo memroos`.

## Operator Representation

- `MEMSEC-01`: API/backend-visible. Admins can list vault artifacts through `GET /api/admin/vault` and replay one through `GET /api/admin/vault/:artifactId/replay`.
- `MEMSEC-02`: Backend/security-policy-visible. Label columns exist on all label-bearing tables with fail-closed defaults and are ready for Phase 75/76 UI and policy surfaces.
