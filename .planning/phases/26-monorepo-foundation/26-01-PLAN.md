# Phase 26 Plan: Monorepo Foundation

## Objective

Make `agent-kitchen` the canonical monorepo without changing the production behavior users already rely on.

## Scope

- Move the Next.js Kitchen dashboard into `apps/kitchen`.
- Move service runtimes into `services/`.
- Preserve root governance files, planning docs, MCP config, data paths, and deployment entrypoints.
- Keep private Knowledge Hub content outside this repo.
- Keep root scripts as simple delegates into `apps/kitchen`.

## Tasks

1. Establish the canonical monorepo directory layout.
2. Update root scripts and app-local package metadata.
3. Make runtime file resolution root-aware for SQLite, config files, and service catalogs.
4. Import memory service files into `services/memory`.
5. Move the existing voice server into `services/voice-server`.
6. Verify Kitchen build, tests, and production startup still work after the move.

## Validation

- `npm --prefix apps/kitchen run test -- --run`
- `npm run build`
- Root `.mcp.json` path inspection
- Production LaunchAgent restart on port `3002`
