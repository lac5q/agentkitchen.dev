<objective>
Implement the Python-side HIL edit-and-continue capability: a validated `PATCH /hil/{id}/edit` FastAPI endpoint that patches a paused LangGraph thread's checkpoint state and audits the change. This plan delivers the orchestration-service half of HIL-01..03; the TypeScript route, client, and UI complete them in Plan 05.

Purpose: Operators currently can only approve/reject a paused HIL task — they cannot correct task state. RESEARCH.md establishes the exact, source-verified pattern: `update_state(config, patch, as_node="route_policy")` so resume re-enters the `needs_approval` conditional and routes to `dispatch` rather than terminating at `END`.
Output: A new `edit_and_checkpoint` runtime method, a schema-validated edit endpoint, and `state_edit` audit lineage rows.
</objective>

  START -> route_policy
  route_policy -[needs_approval conditional]-> "approval" | "dispatch"
  approval -> END        # as_node="approval" would terminate the graph
  dispatch -> END

  runId, taskSummary, requiredCapability, selectedAgentId,
  requiresApproval, status, approvalDecision

Required pattern: graph.update_state(config, patch, as_node="route_policy")
  followed by separate resume() via graph.invoke(Command(resume=decision), config)
