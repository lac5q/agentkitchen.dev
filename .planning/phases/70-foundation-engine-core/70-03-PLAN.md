## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Orchestration service → remote A2A agent | Compensation task dispatched to an untrusted/unverified remote agent |
| Python process → orchestration.db schema | Additive migration mutates persisted schema |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-70-06 | Tampering | compensation logic persistence | mitigate | Compensation stored exclusively as declarative orchestration_lineage rows; process restart cannot lose or corrupt in-memory closures (none exist) |
| T-70-07 | Repudiation | rollback accountability | mitigate | rollback_started/complete + per-hop compensation_done/skipped rows give a full audited rollback trail; orchestration_runs records rollback_reason and rolled_back_at |
| T-70-08 | Denial of Service | unbounded retries | mitigate | RetryPolicy max_attempts is bounded by ORCHESTRATION_RETRY_LIMIT; exhaustion deterministically triggers rollback rather than looping |
| T-70-09 | Tampering | remote agent without compensate capability | accept | A2A v1 has no rollback verb; agents lacking requiredCapability="compensate" yield compensation_skipped — documented safe default (RESEARCH.md Anti-Patterns), no remote rollback trust assumed |
</threat_model>
