<objective>
Introduce the stable `MemoryAdapter` interface and adapter registry in TypeScript, then wrap the three existing memory backends as concrete adapters. This plan delivers MEM-06..08 and is the hard dependency that unblocks Phase 71 recall work.

Purpose: Memory backends are currently called via concrete functions (`searchVectorMemory`, `queryGraphMemory`, `recallByKeyword`). Phase 71 needs a stable, swappable abstraction. RESEARCH.md prescribes a contract-shaped interface — `search()`/`write()`/`health()` only, no client-handle leakage (MEM-P1, mem0 HTTP-only invariant) — plus a registry keyed by `MemoryTier`. The lower-risk integration is the shim approach: existing function signatures stay intact and delegate to the registered adapter when one is present (RESEARCH.md Pattern 5, Pitfall 4).
Output: `adapter.ts` interface, `registry.ts`, three concrete adapters in `backends.ts`, and shim delegation that avoids double-writers.
</objective>
