shown before joining, a `meeting_consents` record, and a consent-gated join
endpoint that mints an opaque `meeting_id` and never persists the room
URL/token to any audit record.

meeting URL/join tokens kept out of `audit_log`. 71-05 delivered the listener
pipeline and the bot-side secrets discipline; this plan delivers the
operator-facing consent gate and the consent-enforcing join API.

`/api/meeting/join` route, the `ConsentDialog` component, and a `meetings` page.

<threat_model>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| operator → /api/meeting/join | Operator submits a room URL + token (secrets) and a consent flag. |
| join endpoint → audit_entries | The join writes an audit record; secrets must not cross into it. |
| join endpoint → conversations.db | A consent record is persisted. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-71-19 | Information disclosure | meeting join audit entry | mitigate | D-13: the audit entry's `entity_id` and `metadata_json` reference an opaque `meeting_id` and label only; the route never passes `roomUrl`/`token` into `writeAuditEntry`. route.test.ts asserts the secrets are absent from the audit row. |
| T-71-20 | Information disclosure | meeting_consents table | mitigate | The table has no room-URL/token column by design (D-14) — secrets cannot be persisted there even by accident. |
| T-71-21 | Elevation of privilege | unconsented meeting join | mitigate | D-14: the join endpoint returns HTTP 403 unless `consentConfirmed: true` is present; the UI keeps the join button disabled until the consent dialog is confirmed. Consent is enforced server-side, not just in the UI. |
| T-71-22 | Spoofing | join request without auth | mitigate | The route requires `authenticateUser` + `requireRole("operator")`; an unauthenticated or reviewer-level caller cannot start a join. |
| T-71-23 | Information disclosure | room URL/token in client state | mitigate | The meetings page submits `roomUrl`/`token` in the request body and does not render them back or persist them in client storage; they are transient. |
| T-71-SC | Tampering | npm installs | mitigate | No new npm packages; no package-legitimacy checkpoint needed. |

</threat_model>
