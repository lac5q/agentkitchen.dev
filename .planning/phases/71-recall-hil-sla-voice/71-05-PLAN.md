Existing voice-server patterns — reuse, do not reinvent.

transcript_writer.py — TranscriptWriter(db_path, session_id).write(role,
content): opens a fresh sqlite3 connection per write, sets WAL + busy_timeout,
INSERT OR IGNORE into `messages` (agent_id='voice', project='memroos'). The new
MeetingWriter follows the SAME connection discipline.

transcript_proc_helper.py — build_transcript_proc(session_id) wires a
TranscriptProcessor's on_transcript_update handler to a writer. The Daily
pipeline uses the same TranscriptProcessor + handler approach.

messages table: id, session_id, project, agent_id, role, content, timestamp,
cwd, git_branch, request_id, UNIQUE(session_id, request_id).
hive_actions table: id, agent_id, action_type
(continue|loop|checkpoint|trigger|stop|error), summary, artifacts, session_id,
timestamp.

<threat_model>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Daily.co room → meeting bot | Untrusted meeting audio + participant metadata cross into the bot. |
| meeting bot → conversations.db | The bot writes transcript rows + highlights. |
| env/CLI → meeting bot | Room URL + join token (secrets) enter via env/args. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-71-15 | Information disclosure | Daily room URL / join token | mitigate | D-13: room URL and token are passed only into the `DailyTransport` constructor; never stored on a returned object, never logged, never written to `audit_log`/`audit_entries`/`messages`. `meeting_bot.py` logs the `session_id` only. test_pipeline_daily.py asserts no readable attribute retains them. |
| T-71-16 | Tampering | pipecat version bump | mitigate | `pipecat-ai[daily]>=1.2,<2.0` is range-pinned below 2.0; the bump is gated by a blocking package-legitimacy human checkpoint (pypi verification); Task 4 regression-runs the full suite. |
| T-71-17 | Spoofing | speaker attribution | accept | Speaker labels come from the Daily participant metadata; a malicious participant could mislabel themselves, but the bot is a listener-only memory-ingestion channel — attribution is best-effort, not an authorization signal. |
| T-71-18 | Elevation of privilege | bot speaking into the meeting | mitigate | D-11: the pipeline has no LLM/TTS and no audio `transport.output()` — the bot physically cannot speak or act in the meeting; it is strictly a listener. |
| T-71-SC | Tampering | `pipecat-ai[daily]` install | mitigate | Blocking `checkpoint:human-verify` (gate="blocking-human") verifies `pipecat-ai` on pypi.org and confirms the version + `daily` extra + `DailyTransport` import path before install. Not auto-approvable. |

</threat_model>
