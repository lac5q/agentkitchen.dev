# Phase 34 Plan 02: Authenticated Universal REST Write API

<objective>
Expose framework-agnostic REST write endpoints backed by the canonical registry service, with per-agent API key authentication enforced consistently.
</objective>

<threat_model>
## Threat Model

- Threat: unauthenticated agents mutate heartbeat, memory, skills, or tool outcome state. Severity: high. Mitigation: shared bearer-key auth helper; route tests cover missing and invalid key for every write endpoint.
- Threat: memory/tool payloads include sensitive data that is reflected back in error responses. Severity: high. Mitigation: return generic error bodies and avoid echoing payload content on failures.
- Threat: route handlers drift by reimplementing auth differently. Severity: medium. Mitigation: centralize auth extraction/verification in the registry service or a small route helper.
</threat_model>
