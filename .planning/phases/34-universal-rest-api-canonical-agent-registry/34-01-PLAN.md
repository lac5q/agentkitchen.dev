# Phase 34 Plan 01: Canonical Registry Schema, Service, And API Keys

<objective>
Create the SQLite-backed canonical agent registry model and service layer that all Phase 34 and future Phase 35 registration adapters will use.
</objective>

<threat_model>
## Threat Model

- Threat: plaintext API keys are stored, logged, or returned from list/get endpoints. Severity: high. Mitigation: generate one-time plaintext keys, store SHA-256 hashes only, redact normal DTOs, and test DB rows do not contain the plaintext key.
- Threat: an attacker spoofs `agentId` in the request body. Severity: high. Mitigation: authentication returns the canonical agent from the bearer key; body IDs are hints only and must match authenticated identity when present.
- Threat: deregistration erases audit history needed for incident review. Severity: medium. Mitigation: soft-delete agents with `deregistered_at` and revoke keys.
</threat_model>
