# Phase 34 Plan 03: Agent Registry UI And Dynamic Flow Roster

<objective>
Add the Kitchen Agent Registry page and migrate Kitchen Floor and Flow roster displays to canonical registry data with zero hardcoded agent identifiers.
</objective>

<threat_model>
## Threat Model

- Threat: one-time API keys remain visible in normal UI state or are refetched later. Severity: high. Mitigation: show key only from registration response, never from list/get DTOs, and label it as one-time.
- Threat: UI deregistration accidentally deletes history. Severity: medium. Mitigation: route calls soft deregistration and UI copy/action state says deregister rather than purge.
- Threat: stale hardcoded Flow IDs imply agents exist when they do not. Severity: medium. Mitigation: static grep and Flow tests remove named roster constants.
</threat_model>
