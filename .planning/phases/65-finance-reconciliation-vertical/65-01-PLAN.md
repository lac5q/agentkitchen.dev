# Phase 65 Plan 01: Finance Reconciliation Vertical

## Product Goal

Bank transaction reconciliation governance runs on Memoroos: transaction events
feed the L3 scorer, reconciliation golden sets power evals, finance terminology
is configurable, and a mock 100-transaction demo writes immutable audit evidence
and HIL exceptions.

## Requirements Covered

- FIN-01: bank transaction CSV/webhook adapter
- FIN-02: reconciliation golden sets and drift validation
- FIN-03: configurable finance terminology

## Implementation Plan

1. Add RED tests for finance reconciliation normalization.
   - Parse CSV rows and webhook payloads into a canonical transaction event.
   - Persist normalized rows to `business_outcome_events`.
   - Write audit entries for match/mismatch/escalation decisions.
   - Open HIL escalations for exception rows.

2. Add finance reconciliation domain modules.
   - Create `apps/kitchen/src/lib/finance-reconciliation/`.
   - Implement CSV parser, webhook normalizer, demo generator, and processing
     service.
   - Keep source payloads in metadata/raw JSON and avoid PII in test fixtures.

3. Wire L3 finance persistence.
   - Reuse `business_outcome_events` with `source_system='finance'`.
   - Store KPI values for matched, mismatched, escalated, and amount-delta
     signals keyed by `correlation_id`.
   - Keep writes idempotent for repeated CSV imports.

4. Wire audit and HIL.
   - Append immutable audit entries for every reconciliation decision.
   - Create HIL escalations for transaction exceptions and unresolved mismatch
     decisions.
   - Include source row identifiers and correlation IDs in audit metadata.

5. Add API/demo entrypoints.
   - Add a finance reconciliation route for webhook ingestion and demo runs.
   - Return summary counts for processed, matched, mismatched, escalated, audit
     entries, and open exceptions.

6. Add finance terminology config.
   - Extend `memroos.eval.yaml` with finance mode labels and threshold defaults.
   - Expose a helper that maps generic UI terms to finance-specific vocabulary
     when enabled.

7. Add reconciliation golden sets.
   - Add match, mismatch, escalation, duplicate, amount-delta, and policy-leak
     examples.
   - Add a deterministic validation script/test that proves drift agreement
     threshold coverage.

8. Update UI surface.
   - Add finance/reconciliation labels where existing business-ops/eval pages
     read terminology config.
   - Add a compact reconciliation demo panel if the existing Business Ops page
     has an appropriate operations surface.

9. Update docs and planning state.
   - Write a phase summary with verification evidence.
   - Update ROADMAP/STATE when phase execution is complete.
