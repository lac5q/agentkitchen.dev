# Phase 66 Plan 01: Self-hosted Hardening + Compliance Posture

## Product Goal

Memoroos runs fully self-hosted with zero external data egress in data-residency
mode, supports local Ollama/vLLM judge endpoints, and exposes production-ready
admin controls for compliance operations.

## Requirements Covered

- INFRA-01: self-hosted full-stack posture
- INFRA-02: local-only data residency mode

## Implementation Plan

1. Add RED tests for data-residency routing.
   - External judge providers are blocked when data residency is enabled.
   - Ollama and OpenAI-compatible local endpoints are accepted.
   - Blocked external calls write audit/security evidence.

2. Add compliance/runtime config.
   - Extend eval/runtime config with `compliance.data_residency.enabled`,
     `judge.local_endpoint`, `judge.provider`, and allowed-host controls.
   - Add `.env.example` entries for local judge and self-hosting flags.

3. Harden judge routing.
   - Route judge calls through a single provider policy helper.
   - Support Ollama and vLLM/OpenAI-compatible local endpoints without
     provider-specific scoring branches.
   - Preserve deterministic test mode for CI and local verification.

4. Add admin compliance controls.
   - Add APIs for user listing, API key rotation, adapter enable/disable, and
     audit-retention policy.
   - Enforce admin role at the handler layer.
   - Append audit entries for sensitive admin changes.

5. Update admin UI.
   - Add a compliance/admin panel or extend the existing admin surface.
   - Show data-residency status, judge endpoint status, adapter status, and
     retention settings.

6. Update compose/setup posture.
   - Ensure compose has local service options for mem0, Neo4j, SQLite, and a
     local vector store posture when data residency is enabled.
   - Keep Qdrant Cloud default behavior documented for non-residency mode.
   - Add setup validation warnings for missing local judge endpoints.

7. Add docs and planning summary.
   - Document data-residency behavior, supported local judge endpoints, and
     admin control audit evidence.
   - Record verification results and any deferred full-container checks.
