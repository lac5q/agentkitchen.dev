<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| filesystem → app | ~/.memroos/context-sources.local.json is read from user home; attacker with home-dir write access can inject source config |
| env → app | CONTEXT_SOURCES_LOCAL_CONFIG path is operator-controlled; misconfigured path could cause silent fallback to base config |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-99-01 | Tampering | context-sources.local.json (home dir) | accept | Same trust level as repo config; both require filesystem access. Home dir is operator-controlled — no additional mitigation warranted. |
| T-99-02 | Information Disclosure | context-sources.local.json committed to git | mitigate | Gitignore entry in .gitignore prevents accidental commit. Example file ships patterns without real credentials. |
| T-99-03 | Denial of Service | malformed local JSON silently swallowing errors | mitigate | loadContextSourceContracts() will surface JSON.parse() exceptions naturally — no catch needed; fail-loud is correct behavior. |
| T-99-SC | Tampering | npm/pip/cargo installs | accept | No new package installs in this phase; os module is Node.js built-in. |
</threat_model>
