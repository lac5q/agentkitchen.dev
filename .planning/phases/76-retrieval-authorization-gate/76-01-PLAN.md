# Phase 76 Plan 01 — Retrieval Authorization Gate

## Goal

Introduce a shared memory-use authorization decision point and apply it first to the highest-risk local retrieval and derived-index paths.

## Requirements

- MEMSEC-04

## Implementation Plan

1. Add `lib/memory/policy-gate.ts`.
   - Inputs: actor identity/role/capability, purpose, tenant/project hints, and label snapshot.
   - Outputs: `allow`, `deny`, `redact`, or `review-required` plus reason.
   - Default missing labels to `visibility=private`, `policy=sealed`.

2. Apply to local recall.
   - Filter `/api/recall` BM25, semantic, hybrid, and agent-history results before response serialization.
   - Only update recall side effects (`recall_log`, salience `access_count`) for allowed rows.
   - Write denied/review-required decisions to `audit_log`.

3. Apply to adapter-backed episodic search.
   - `EpisodicMemoryAdapter.search()` filters `recallByKeyword()` rows through the same gate.

4. Apply to derived embedding writes.
   - `messagesNeedingEmbedding()` only returns rows with promoted `policy='indexable'` and non-private visibility.

5. Remaining Phase 76 work.
   - Apply the same gate to `/api/memory/search`, `/api/memory/multi-search`, ChatGPT Actions retrieval/use, export/summarization, dispatch context assembly, and evidence bundle reads.
   - Add tests for vector/graph/mem0 HTTP content written outside the TypeScript ingest path.

## Verification Plan

- Policy-gate unit tests.
- Recall route tests.
- Embedding store/job tests.
- Typecheck, lint, and whitespace checks.
