# Phase 75 Plan 01 — Classification Cascade + Ingestion Gate

## Goal

Classify every newly vaulted artifact with fail-closed security labels before it can leave the raw vault, and route sensitive, uncertain, or public-promotion cases into a visible human-review workflow.

## Requirements

- MEMSEC-03
- CTX-FOLLOWUP-03

## Implementation Plan

1. Add a classification review schema.
   - Create `classification_reviews` with artifact pointer, proposed label dimensions, reason codes, evidence spans, review decision fields, and HIL escalation linkage.
   - Keep artifact labels append-only by adding new `artifact_labels` versions rather than mutating prior rows.

2. Add the deterministic-first classifier.
   - Reuse `content-scanner.ts` for PII/secrets/payment/credential signals.
   - Add source metadata and keyword gates for client, legal, finance, HR, personal, engineering, and public-promotion cases.
   - Default every artifact to `visibility=private` and `policy=sealed`; promote only through explicit review decisions.

3. Gate ingestion through classification.
   - After a session artifact is written to the vault, classify it immediately.
   - Stamp the latest label back onto source `messages` rows while preserving the raw vault artifact and label history.

4. Add human-review APIs.
   - `GET /api/classification/reviews` lists open/all review items for reviewer+ roles.
   - `POST /api/classification/reviews/[reviewId]/decision` records approve/deny/redact decisions, appends label versions, updates source labels, resolves the linked HIL item, and writes to `audit_log`.

5. Verify.
   - Classifier unit tests for harmless default, PII/secrets, public-promotion, and vault stamping.
   - API tests for listing review items and recording decisions.
   - Re-run Phase 74 vault tests, typecheck, lint, and whitespace checks.

## Risk Notes

- `initSchema` is CRITICAL blast radius because every DB-backed path calls it through `getDb`; migration is additive and idempotent.
- Ingestion hooks are LOW blast radius and flow through `ingestAllSessions` plus `/api/recall/ingest`.
- External LLM adjudication is intentionally not used for this first gate. Low-confidence or high-stakes cases abstain into human review rather than auto-promoting content.
